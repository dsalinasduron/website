Unit Testing

Unit testing is a very powerful technique for ensuring your code meets requirements and specifications.

As a general rule, you should always run/re-run unit tests whenever you make any change to working code. This demonstrates the power behind unit tests: you feel free to make changes to code as you have an easy and quick way of ensuring your changes did not break any existing, properly-working code.



In most instances thus far, you have been given unit tests that test your implementation of certain methods and classes.

The  area() and perimeter()methods are not yet implemented. However, we will write the unit test first using a practice known as test-driven development (TDD) whereby you write the test before writing the implementation of the method being developed.  TDD is widely practiced in software development and its basic premise is that you really shouldn't write something unless you know how to test it.
