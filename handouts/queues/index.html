Queue

In-class exercise

A queue is a FIFO (first-in first-out) data structure. It operates much like what we call a line, although in the UK they indeed call it a queue.

Queue Interface

- Queue.java

Example Usage

Queue<String> queue = new QueueImpl<String>();

queue.add("Jim");
queue.add("Jess");
queue.add("Jill");
queue.add("Jane");
queue.add("Joe");

String front = queue.remove(); // returns "Jim"
System.out.println(front + " is at the front of the queue.");

front = queue.remove(); // removes and returns "Jess"
System.out.println(front + " is removed from the queue.");

front = queue.remove(); // returns "Jill"
System.out.println(front + " is at the front of the queue.");

Deque


A deque (pronounced "deck") is short for "double-ended queue" where you can insert and delete at either ends of the data structure.

Deque Interface

- Deque.java

Example Usage

After the following statements are performed, what string is at the front of the deque? What string is at the rear?

Deque<String> deque = new DequeImpl<String>();
deque.addToFront("Jim");
deque.addToBack("Jess");
deque.addToFront("Jill");
deque.addToBack("Jane");
String name = deque.getFront();
deque.addToBack(name);
deque.removeFront();
deque.addToFront(myDeque.removeBack());

PriorityQueue

A priority queue is simply a queue with priority. That is, the element with the highest priority is at the front of the queue, not necessarily the element that arrived first. (Although elements that have the same priority may be serviced in FIFO order.)

The Java API provides a priority queue. We will have to follow the inheritance hierarchy and examine the AbstractQueue that PriorityQueue extends.


Example Usage:

- PriorityQueueExample.java

Notice it uses the natural ordering of objects for referring to priority. This natural ordering refers to the Comparable interface.

The implementation of PriorityQueue calls the compareTo() method of objects being inserted into the priority queue to determine order.
