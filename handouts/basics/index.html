How to Override Equals

The following  method overrides equals() for the CircleV1 class:

    public boolean equals(Object other) {
        /**
         * Quick check #1: No object can be equal to null
         */
        if (other == null)
            return false;

         /**
         * Quick check #2: If 'this' and 'other' are the
         * same object, they are equal so return true.
         */
        if (other == this)
            return true;
       
        /**
         * Quick check #3: Make sure that other
         * is in fact a Circle. If it is not,
         * return false.
         */
        if ( !(other instanceof CircleV1))
            return false;
       
        /**
         * We know that other is a Circle, let's now
         * compare the relevant instance data
         */
       
        /* typecast other to a Circle */
        CircleV1 o = (CircleV1)other;
       
        /**
         * Check the instance data for equality
         */
        if ( (this.radius == o.radius) && (this.color.equals(o.color)) )
            return true;
        else
            return false;
  
    }
Add this method to CircleV1.java and re-run the unit tests.

Notice how we use == for primitive data types, and .equals() for objects.


We do not need to test all fields for equality. For example, the following class file represents a Process in a computer system:

- Process.java

where a process is represented by a (1) command, (2) unique process id, and (3) state. (Processes run commands (ie '/usr/bin/javac' or "C:\Program Files\javac.exe"), are associated by unique integer values, and have a state (they are either running on a CPU, are ready - but waiting - to run on a CPU, or are blocked such as waiting for input.)

Since process id's are unique, it doesn't make sense for two (or more) processes to have the same process id.
