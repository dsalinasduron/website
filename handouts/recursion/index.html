Simple Example

Output a list of numbers from n down to 0

Iterative Solution

public static void iterative(int n) {
        for (int i = n; i >= 0; i--)
            System.out.println(i + " ");
}

Recursive Solution

    public static void recursive(int n) {
        if (n < 0)
            return; // Base Case
        else {
            System.out.print(n + " ");
            recursive(n - 1); // Recursive Case
        }
    }

The general distinction between iterative and recursive solutions is that an iterative method contains a loop, a recursive method calls itself.

When Not To Use Recursion

- Tail recursion. This occurs when the last line of the method is a recursive call. A simple iterative loop works just as well as recursion.

- Repeating the same action over and over (Fibonacci Sequence)


When To Use Recursion

- When applying to naturally recursive problems (various sorting algorithms) and data structures (trees) - two topics we will see  this semester.)

- Backtracking


Backtracking

Backtracking is a form of a brute force or exhaustive solution - we potentially try every possibility  when searching for a solution.

General Backtracking Algorithm

public boolean backtracking (solutionSoFar) {
     if (solutionSoFar is final solution)
          return true
     else {
          for each possible choice
               if (backtrack(solutionSoFar + choice)) 
                    return true // this choice returns the final solution
     }
     return false // tried every possible solution without finding a solution
}
