A post-order traversal of a tree visits each node in the tree by visiting all the children of a node before visiting the node itself. By that rule, we know that the root (top of the tree) must be visited last in all preorder traversals. Part 4. Complete the In-Order Method

An in-order traversal of a tree visits the left child, then the parent, then the right child. By that rule, we know that the root (top of the tree) must be visited after all the nodes in the left subtree are visited and before all the nodes in the right subtree are visited. You may assume that, if a node has only one child, it is a left child. For example, a preorder traversal of the tree above visits nodes in the following order: Q, D, G, L, H, F, T, R, M.

Part 5. Implement your own UndirectedGraph class.
You must implement the UndirectedGraph.java interface. Your class must be able to represent a graph and answer some questions about it. That means it must keep track of the vertices and edges in the graph. You are free to choose which classes to use to represent a vertex and an edge. (See this example. You may write your own Vertex and Edge class. I recommend this since it will help you make the methods easy to implmement. The interface has four methods:
  • getVertices() returns all the vertices of the graph inside of a list.
  • getEdges() returns all the edges of the graph inside of a list.
  • isComplete() return true if the number of edges is the maximum possible. For an undirected graph, this is n * (n - 1) / 2, where n is the number of vertices.
  • hasCycle() returns true if there is a cycle in the graph. Remember that a cycle is a path that intersects itself (forms a circle).
