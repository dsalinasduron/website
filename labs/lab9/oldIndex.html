Part 5. Implement getHeight() Method

Just as getSize() can be implemented recursively, there is a similar approach for determining the height of a binary tree:
int getHeight(node)
{
    if node == null
        return 0
    else {
        maxHeight = max( getHeight(left child of node), getHeight(right child of node) )

        return maxHeight + 1
    }
}

However, the above algorithm has one small bug in it (hint - we measure the height of a tree beginning at zero) that you will need to fix.

Just as is with earlier methods, you may find the above algorithm more useful as a private helper method that is called by the public getHeight() method defined in BinaryTree.


Part 6. Implement a recursive algorithm for getSize()

One way of keeping track of the number of elements in the tree (its size) is by using a counter that is incremented whenever a new element is  added to the tree. However, this approach of using a counter is awkward and can instead be performed by using a recursive algorithm.

You may notice that the API in BinaryTree for getSize() is not passed a parameter. You may find that it is easiest to instead create a private helper method that is based on the  algorithm for getHeight(), and have the getsize() method that is part of the interface call this helper method.

Be sure to test your implementation of getSize() .
