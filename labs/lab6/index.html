Part III - Testing Palindromes with Stacks

A palindrome is a phrase that is spelled the same both forwards and backwards. For example

OTTO

MADAM 

110202011

are palindromes. 

OTHELLO

MADAMX

110202012

are not palindromes.

Working with your partner, devise an algorithm that uses stacks (hint - you'll likely need two) to determine whether a given phrase is a palidrome or not. Your program should be case-sensitive (so OTto is not a palindrome) and it should examine all characters in the String (so "dennis sinned" is a palindrome, but "never odd or even" is not a palindrome.)

If you are struggling with your algorithm, review the stack algorithms in the class notes, in particular the balanced delimiters example.


You should begin work using the following program:

- PalindromeChecker.java

There are no unit tests for this portion of the lab. However, be sure to carefully test your palindrome checker.

Extra Credit

A binary palindrome is a number whose binary representation is a palindrome. For example, the number 9 in decimal (base 10) is 1001 in binary (base 2). The binary representation 1001 is a palindrome. However, the decimal number 41 in binary is 101001, which is not a palindrome.

Write a Java program IsBinaryPalindrome.java that is passed a decimal number on the command line  (Run As -> Run Configurations -> Arguments) and outputs whether or not the decimal number is a binary palindrome.

This will first involve understanding how binary numbers are represented. Once you have a basic understanding, you will need to devise an algorithm that converts a decimal number to its binary representation.
