It should maintain the side (heads/tails) using an enumerated data type:

public enum CoinSide {HEADS, TAILS};


For your instance variable, use this enumeration type CoinSide. You should declare a CoinSide instance variable in the same way that you would declare any other instance variable. However, this object can only have two values, CoinSide.HEADS and CoinSide.TAILS.

Example Usage:

public enum CoinSide {HEADS, TAILS};    // CoinSide is a new data type
CoinSide side;                          // side is of type CoinSide

. . .

side = CoinSide.HEADS;                  // sets side to heads

. . .

if (side == CoinSide.TAILS)
{
    // some logic is side is set to tails
}

 
This class should include the following methods:

    * a constructor that initializes  the coin at heads

    * three mutators:
          o void setToHeads() - sets the coin to heads
          o void setToTails() - sets the coin to tails
          o void toss() - sets the coin randomly to heads or tails. Use java.lang.Math.random() to set the coin to heads 50% of the time, and tails 50% of the time.
            (If you've never used java.lang.Math.random() before, read more about it in the Java API.)

    * isHeads() - returns true if the coin is heads
    * isTails() - returns true if the coin is tails
    * toString() - returns the String "heads" or "tails" based on the side coin

Make sure your class compiles and you implement all of the above requirements.

Comment your code as you proceed - it is generally easier to comment code while you write it, versus after you write it.

Step 4: Comment your code

You should have commented each of your classes as you completed them. Good programming style includes:

    * using meaningful variable names
    * using proper indentation
    * including a comment block before the program that describes what program does and includes your name and the assignment
    * including a comment block before each public method that describes what the method does and what its preconditions (assumptions) are
    * including comments anywhere the code may be confusing


Extra Time? Write a game.

Write a program in a new class: CoinGame.java. You should ask the user to guess whether the coin will be heads or tails. Then flip the coin and report if the coin was heads or tails. Repeat until the user is done playing.

The java.util.Scanner class is a good choice for capturing user input.

After the user is done, report the number of coin tosses, and how many times the user guessed correctly. Also include the percentage of correct guesses.
