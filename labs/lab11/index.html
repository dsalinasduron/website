Part II - Double Hashing

Create a class named HashDictionaryDouble.java - you can use your class file from Part I as a starting point. You will implement double hashing in this class so that it passes the following unit tests:

- TestDouble.java

Double hashing works by creating a second hash function and using that value as a probe in the case of a collision.  The algorithm is shown in the notes on hashing. (Think of how the probeValue() helper method in HashDictionaryLinear.java can be used to implement this.)

Use the following technique for calculating the second hash value:

(1 + (Math.abs(key.hashCode())) % (currentCapacity - 2) )

(Although you are of course welcome to try a different approach.)

You will have to support all of the functionality described in the section on linear probing.

Make sure your implementation passes all unit tests before proceeding to the next step!


Part III - Separate Chaining

The third approach for implementing a dictionary involves separate chaining, where each element in the array is a List of TableElement objects:

private List<TableElement<K, V>>[] dictionary;

The List you will use is the LinkedList that you created in Lab 4. This will require modifying that solution so that it now uses generics. Below is the List interface with generics:

- List.java

Modify your LinkedList.java file for Lab 4 so that it now uses generics and passes the following unit test:

- TestLinkedList.java

Next, create a file named HashDictionaryChained.java that implements DictionaryInterface.


The approach for using separate chaining is a bit different than with open addressing (linear probing or double hashing). For one, a remove() operation can remove the [key:value] mapping from the linked list. Also, it is unnecessary to rehash the list if it exceeds a certain threshold. In open addressing rehashing was necessary, not only for performance, but also to guarantee there was a spot in the array to map a new [key:value] pair.

Several of the methods will require accessing the elements in the linked list for a given hash location (known as a hash bucket).

The following unit test will be used to test your implementation:

- TestChained.java
