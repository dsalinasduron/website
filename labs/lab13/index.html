Testing Your Priority Queue

To motivate testing of your priority queue, we will use an example from operating systems: An operating system is responsible for maintaining all running programs in your system - the operating system refers to running programs as tasks. A task has several characteristics, for our example we will identify a task as having a name and a priority. The name refers to the name of the running program (i.e. Chrome), the priority refers to a integer value that specifies the relative priority of the task, where a higher numeric priority indicates a higher relative priority. An operating system will schedule the task with the highest priority to run on an available processing core.

Implement HeapSort

A heap can also be used to implement a heap sort algorithm. We went over the general algorithm in class (for a minimum heap):

1. Add all items that need to be sorted into a min heap.

2. Remove the items from the heap, and insert them into the sorted array from lowest to highest.

We are using a max heap - not a min heap, but it should be clear how you can use a max heap to implement the heap sort.
